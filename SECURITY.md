# Security Policy

Public GitHub issues and pull requests are welcome for normal bugs,
improvements, and contributions.

## Reporting a vulnerability

If you believe you found a security vulnerability, please report it privately
instead of opening a public issue.

Report security issues privately by email to:

- support@a11ymind.ai

Please include, if possible:

- a short description of the issue
- affected routes, pages, or components
- reproduction steps or proof of concept
- impact assessment if known

We will review reports as quickly as possible and coordinate a fix before any
public disclosure.

## Scope

This policy covers the a11ymind web application and the code in this
repository, including:

- authentication flows
- billing and Stripe integration
- scan routes and background jobs
- dashboard and reporting features

Third-party services and infrastructure providers are out of scope except where
the issue is caused by how this application integrates with them.
